Project: Offline Public Transport Delay Attribution System
City: Shivamogga

Objective
Analyze historical bus schedule data and attribute delays
to likely operational causes.

System Features
• Works completely offline
• Uses historical schedule vs actual timing
• Rule-based delay attribution
• Generates route-wise and stop-wise reports

Input Parameters
1. Route ID
2. Trip ID
3. Stop ID
4. Scheduled arrival time
5. Actual arrival time
6. Scheduled departure time
7. Actual departure time

Output
1. Route Delay Report
2. Stop Delay Report
3. Delay Cause Classification
